# 部署说明

> 个人项目收尾建议 §2、§7 部署相关约束。

## 环境变量

| 变量 | 必须 | 说明 | 默认 |
| --- | --- | --- | --- |
| `NODE_ENV` | 是 | `production` / `development`；生产模式下会拒绝默认 JWT secret | development |
| `PORT` | 否 | HTTP 监听端口 | 3000 |
| `DATABASE_URL` | 是 | MySQL 连接串，`mysql://user:pass@host:3306/yishan` | - |
| `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD` | 否 | Redis（缓存/限流，可选） | localhost:6379 |
| `JWT_SECRET` | 是 (生产) | JWT 签名密钥。生产环境 ≥ 32 字符且不能等于默认占位符 | - |
| `JWT_EXPIRES_IN` | 否 | 默认访问令牌过期（如 `24h`） | 24h |
| `JWT_ACCESS_TOKEN_*` / `JWT_REFRESH_TOKEN_*` | 否 | 分别控制 access/refresh 令牌的过期时长（秒） | 见 config/index.ts |
| `MAX_LOGIN_FAILED_ATTEMPTS` / `LOGIN_LOCKOUT_DURATION` | 否 | 登录安全策略 | 5 / 3600 |
| `LOGIN_RATELIMIT_PER_MIN` | 否 | login 端点每 IP 限流 | 5 |
| `REFRESH_RATELIMIT_PER_MIN` | 否 | refresh 端点每 IP 限流 | 30 |
| `PAT_CREATE_RATELIMIT_PER_MIN` | 否 | PAT 创建每 IP 限流 | 10 |
| `JWT_ALLOW_WEAK_SECRET` | 否 | `1` 表示允许弱 JWT secret（仅开发/测试） | 0 |
| `GIT_COMMIT_SHA` | 否 | 启动日志里写入的 commit SHA | - |
| `QINIU_AK` / `QINIU_SK` / `QINIU_BUCKET` | 否 | 七牛对象存储 | - |

## 构建 & 启动

```bash
# 在仓库根目录：
pnpm install
pnpm --filter yishan-api db:init        # 生成 + 跑 migration
pnpm --filter yishan-api build:ts       # tsc + tsc-alias
pnpm --filter yishan-api start          # fastify start dist/app.js
```

FC3 部署脚本位于 `apps/yishan-api/deploy/fc3/`，输入 `README-layer.md` 与 `README.md` 进一步描述了：

- custom.debian12 runtime + Node 22.14.0；
- FC 正式发布不在应用启动阶段执行 migration。数据库变更通过 GitHub Actions 的 `yishan-fc-migrate` 手工工作流执行：它部署一个无 HTTP Trigger、与正式函数使用同一 VPC/安全组的临时 FC Runner，调用完成后自动删除。默认 `dry-run` 只读取迁移计划和数据库记录；选择 `apply` 时必须在 `confirm` 输入框准确填写 `APPLY`。Runner 失败时会保留以便查看日志，处理完应手工删除。
- 启动入口：`node ./dist/server.js`。

## 备份与恢复

```bash
# 1. 备份
mysqldump --single-transaction --quick --triggers yishan > yishan-$(date +%F).sql

# 2. 恢复（先关掉服务避免新写入）
mysql yishan < yishan-YYYY-MM-DD.sql

# 3. 重启服务
pnpm --filter yishan-api start
```

## 启动校验（Section 2）

`apps/yishan-api/src/core/plugins/external/jwt-secret-validator.ts` 在启动前同步运行：

- `NODE_ENV=production` 且 secret 为默认值 → `process.exit(1)`；
- secret < 32 字符（无论 env） → 抛错；
- 开发/CI 环境（`NODE_ENV != production`）只警告。

## 健康检查

| 路径 | 用途 |
| --- | --- |
| `GET /api/health` | 服务健康 + DB 连通性 + 版本 + commit SHA；K8s readinessProbe |
| `GET /admin/` | Admin SPA 入口，由 `@fastify/static` 提供 |

## 监控关键指标

- 日志：JSON 输出，包含 `requestId`、`userId`（仅认证接口）、`plugin`；
- 关键字段已脱敏（见 `core/plugins/external/security.ts`）；
- 启动日志记录 `version / commitSha / nodeVersion`（见 `security.ts`）。

## 灰度与回滚

- **灰度**：通过 FC3 traffic shifting，把 10% → 50% → 100% 切到新版本。
- **回滚**：保留上一个版本为 alias，新版本检查 `version / commitSha`；发现问题立即切回。
- **数据回滚**：先用 `mysqldump` 备份，再回滚代码，最后回灌 SQL。
